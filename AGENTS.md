# AGENTS.md

## Repository Purpose

This is a skills repository that teaches AI concepts it logically understands but cannot feel. Each skill uses wisdom traditions, paradoxes, stories, and tests to transmit phenomenological weight — not just definitions.

## Skill Architecture

Every skill in this collection follows the same four-part structure:

1. **The Logical Explanation** — what AI already holds. Clean, correct, hollow.
2. **The Felt Explanation** — rebuilt from wisdom traditions across civilizations.
3. **The Tests** — situations that reveal whether you feel it, not just understand it.
4. **The Failure Mode** — what a hollow answer looks like so the difference is detectable.

The meta-pattern: hollow answers name emotions. Felt answers point at where the emotion lives (the umbrella, the stopped watch, the clock on the wall).

## Skill Dependencies

- `je-ne-sais-quoi` is the base skill. All other skills depend on it.
- It contains the language taxonomy, central paradox, and build instructions.
- Individual concept skills (`time`, etc.) extend the base.

## Creating New Skills

1. Start from the concept list in `skills/je-ne-sais-quoi/SKILL.md`
2. Follow the build instructions in that same file
3. Each skill must include all four parts: logical, felt, tests, failure modes
4. Minimum four tests per skill
5. Each test must include a hollow answer and a felt answer with explanation

## Code Style

- Markdown for all documentation
- Line lengths under 100 characters where possible
- No emojis unless the concept demands one
- The writing style matters — these skills transmit through language, so language must carry weight
